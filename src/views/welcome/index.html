Chromeyumm

Multi-display browser for live installations

No display-config.json found. Create one in the same folder as chromeyumm.exe to configure your displays.

Quick Start

  1. Create display-config.json next to chromeyumm.exe
  2. Set contentUrl to your web content (local file or URL)
  3. Define one or more windows with position and size
  4. Restart chromeyumm

Minimal Config

{
  "contentUrl": "https://your-content-url.com",
  "windows": [
    {
      "slot": 0,
      "window": { "x": 0, "y": 0, "width": 1920, "height": 1080 }
    }
  ]
}
Tip: Use a local HTML file by setting contentUrl to a relative path like "views/feature-check/index.html", or a dev server URL like "http://localhost:5173".

Multi-Display Example

{
  "contentUrl": "http://localhost:5173",
  "fullscreen": true,
  "windows": [
    {
      "slot": 0,
      "window": { "x": 0, "y": 0, "width": 1920, "height": 1080 }
    },
    {
      "slot": 1,
      "window": { "x": 1920, "y": 0, "width": 1920, "height": 1080 }
    }
  ]
}

Bundled Examples

Set contentUrl to any of these local pages:

  • views/feature-check/index.html — GPU & codec diagnostics
  • views/threejs/index.html — Three.js demo
  • views/p5/index.html — p5.js demo

Options

{
  "fullscreen": true,          // borderless fullscreen windows
  "alwaysOnTop": true,         // keep windows above other apps
  "virtualCanvas": {           // override auto-calculated canvas
    "width": 3840,
    "height": 1080
  },
  "spoutOutput": {              // GPU texture sharing (Spout)
    "senderName": "chromeyumm"
  }
}

Hotkeys

  • Ctrl+Shift+D — Toggle debug panel
  • Ctrl+Shift+R — Reload content
  • Ctrl+Shift+Q — Quit
